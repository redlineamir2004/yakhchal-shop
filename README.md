# راهنمای راه‌اندازی اپ کنترل انقضا محصولات

این اپ آماده‌ست، فقط چند مرحله باقی مونده تا فعالش کنید. کل مراحل حدود ۱۵-۲۰ دقیقه طول می‌کشه و همه‌چیز **رایگانه**.

---

## مرحله ۱: ساخت پروژه Firebase (دیتابیس مشترک + لاگین)

1. برید به [console.firebase.google.com](https://console.firebase.google.com) و با جیمیل وارد بشید.
2. **Add project** بزنید، یه اسم بدید (مثلاً `yakhchal-shop`) و پروژه رو بسازید. (Google Analytics رو می‌تونید غیرفعال بذارید، لازم نیست.)
3. داخل پروژه، از منوی سمت چپ **Build > Authentication** رو باز کنید → **Get started** → تب **Sign-in method** → گزینه‌ی **Email/Password** رو فعال کنید.
4. از منوی سمت چپ **Build > Firestore Database** رو باز کنید → **Create database** → حالت **Production mode** رو انتخاب کنید → یه لوکیشن نزدیک (مثلاً `europe-west` یا `asia-*`) انتخاب کنید.
5. برید به تب **Rules** همون Firestore و محتوای زیر رو جایگزین کنید و **Publish** بزنید:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    function isSignedIn() { return request.auth != null; }
    function isAdmin() {
      return isSignedIn() &&
        get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
    }
    match /users/{userId} {
      allow read: if isSignedIn();
      allow create, update, delete: if isAdmin();
    }
    match /products/{barcode} {
      allow read, write: if isSignedIn();
    }
    match /batches/{batchId} {
      allow read, write: if isSignedIn();
    }
  }
}
```

6. برگردید به صفحه اصلی پروژه (⚙️ کنار Project Overview → **Project settings**). پایین صفحه، بخش **Your apps** → آیکون `</>` (Web) رو بزنید → یه نام دلخواه بدید → **Register app**.
7. بعد از زدن **Register app**، صفحه‌ای باز میشه با چند تب بالاش (معمولاً `npm` / `CDN` / **`Config`**). روی تب **Config** کلیک کنید — این همون گزینه‌ایه که کد ساده و آماده (بدون نیاز به نصب چیزی) رو نشون میده. یه چیزی شبیه این می‌بینید:

```js
const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "yakhchal-shop.firebaseapp.com",
  projectId: "yakhchal-shop",
  storageBucket: "yakhchal-shop.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abcdef123456"
};
```

⚠️ این کد رو **داخل خود سایت Firebase تغییر ندید** — فقط از همونجا کپی‌اش کنید (روی خط‌ها می‌تونید تایپش کنید یا دکمه‌ی کپی کنار کادر رو بزنید، اگه بود).

حالا برید سراغ فایلی که من براتون ساختم به اسم **`firebase-config.js`** (داخل پوشه‌ی `expiry-app` که دانلود کردید):

1. فایل `firebase-config.js` رو با یه ادیتور متن ساده باز کنید (مثلاً Notepad در ویندوز — کلیک راست روی فایل → Open with → Notepad).
2. داخلش یه آبجکت مشابه با مقادیر `PASTE_YOUR_API_KEY_HERE` و بقیه می‌بینید.
3. کل آبجکتی که از سایت Firebase کپی کردید (مرحله ۷) رو جایگزین همون آبجکت داخل فایل کنید — یعنی کل بلوک `const firebaseConfig = {...}` رو با نسخه‌ی واقعی خودتون عوض کنید، نه اینکه تک‌تک مقادیر رو دستی جایگزین کنید (ساده‌تر و بدون خطاست).
4. فایل رو **Save** کنید (همون فرمت `.js` بمونه، چیزی به اسمش اضافه نشه).

بعد از این کار، فایل `firebase-config.js` شما دقیقاً به پروژه‌ی Firebase خودتون (`yakhchal-shop`) وصل میشه.

## مرحله ۲: ساخت اولین حساب ادمین

چون اپ فقط صفحه‌ی لاگین داره (بدون ثبت‌نام عمومی)، اولین حساب ادمین رو باید دستی بسازید:

1. در Firebase Console → **Authentication** → تب **Users** → **Add user** → ایمیل و رمز خودتون رو بزنید → **Add user**. یه `User UID` براش ساخته میشه، همون رو کپی کنید.
2. برید به **Firestore Database** → **Start collection** → نام کالکشن: `users`.
3. Document ID رو دقیقاً همون `User UID` که کپی کردید بذارید.
4. سه فیلد اضافه کنید:
   - `name` (string) → اسم شما
   - `email` (string) → همون ایمیلی که ساختید
   - `role` (string) → مقدار `admin`
5. Save کنید.

حالا می‌تونید با همین ایمیل/رمز وارد اپ بشید و از داخل اپ (تب «کارمندان») حساب بقیه کارمندها رو اضافه کنید — دیگه نیازی به دستکاری دستی Firebase نیست.

## مرحله ۳: تست اپ (قبل از تبدیل به APK)

فایل‌های اپ رو نمی‌تونید مستقیم با دوبار کلیک روی `index.html` باز کنید (مرورگرها اجازه دسترسی به دوربین رو در حالت `file://` نمی‌دن). برای تست سریع:

- پوشه‌ی اپ رو در یه هاست رایگان بذارید — ساده‌ترین راه: **GitHub Pages**.
  1. وارد [github.com](https://github.com) بشید (اگه حساب ندارید، یکی رایگان بسازید).
  2. بالا سمت راست صفحه، روی علامت **+** کلیک کنید → **New repository**.
  3. یه اسم بدید (مثلاً `expiry-app`)، گزینه **Public** انتخاب‌شده بمونه، و روی **Create repository** بزنید.
  4. بعد از ساخته شدن ریپازیتوری، وارد صفحه‌ی خالی همون ریپو میشید. اونجا دو تا راه دارید:
     - **راه ساده‌تر (پیشنهادی):** فایل‌ها و پوشه‌ی `expiry-app` رو که روی کامپیوترتون دارید، مستقیم با ماوس **درگ کنید و روی وسط همون صفحه‌ی GitHub رها کنید** (Drag & Drop). خودش می‌گه "Drop to upload your files".
     - **یا:** بالای لیست فایل‌ها (سمت چپ، کنار دکمه‌ی سبز `<> Code`) دنبال دکمه‌ای به اسم **Add file** بگردید → از منوش **Upload files** رو بزنید.
     - ⚠️ اگه هیچ‌کدوم از این دکمه‌ها رو پیدا نکردید، probably روی صفحه‌ی اصلی گیت‌هاب (نه صفحه‌ی ریپوی خودتون) هستید — مطمئن بشید اول وارد ریپوی `expiry-app` که ساختید شدید (از منوی بالا، روی اسم پروفایلتون → Your repositories → روی اسم ریپو کلیک کنید).
  5. بعد از انتخاب/رها کردن فایل‌ها، پایین صفحه یه دکمه‌ی سبز **Commit changes** میاد، روش کلیک کنید تا آپلود نهایی بشه.
  6. برید به تب **Settings** بالای ریپو → از منوی سمت چپ **Pages** رو بزنید → زیر «Branch»، به‌جای `None`، گزینه‌ی `main` رو انتخاب کنید → **Save**.
  7. بعد از حدود ۱-۲ دقیقه، بالای همون صفحه یه آدرس سبزرنگ مثل `https://username.github.io/expiry-app/` ظاهر میشه. همون رو با گوشی باز کنید و تست کنید.

## مرحله ۴: تبدیل به فایل APK

بعد از اینکه اپ روی آدرس بالا کار کرد:

1. برید به [pwabuilder.com](https://www.pwabuilder.com)
2. آدرس هاست‌شده‌تون (مثلاً `https://username.github.io/repo-name/`) رو وارد کنید و **Start** بزنید.
3. بعد از آنالیز، روی **Package for stores** → **Android** کلیک کنید.
4. تنظیمات پیش‌فرض معمولاً کافیه → **Generate** بزنید → یه فایل `.apk` (یا `.aab`) دانلود می‌کنید.
5. فایل APK رو روی گوشی اندروید منتقل کنید و نصب کنید (شاید لازم باشه «نصب از منابع ناشناس» رو در تنظیمات گوشی موقتاً فعال کنید).

بعد از نصب، اپ کاملاً مثل یه اپ نیتیو با آیکون مستقل کار می‌کنه. چون دیتا مشترکه (Firestore)، برای دیدن آخرین تغییرات نیاز به اتصال اینترنت (نت یا وای‌فای) داره؛ رابط کاربری خودش (ظاهر اپ) حتی آفلاین هم لود میشه.

## نکات مهم

- **حریم خصوصی دیتا:** دیتای شما در سرورهای Google/Firebase (پلن رایگان Spark) ذخیره میشه. برای یه فروشگاه با حجم داده‌ی معمولی، سقف رایگان (روزی حدود ۵۰ هزار خواندن/نوشتن) کاملاً کافیه.
- **افزودن/ویرایش/حذف:** طبق درخواستتون، هم ادمین هم کارمند عادی می‌تونن محصول ثبت، ویرایش و حذف کنن. تنها تفاوت ادمین اینه که تب «کارمندان» رو می‌بینه و می‌تونه حساب کارمند جدید بسازه.
- **رنگ‌بندی هشدار:**
  - 🟢 سبز: بیشتر از ۳۰ روز مانده
  - 🟡 زرد: بین ۱۵ تا ۳۰ روز
  - 🟠 نارنجی: بین ۷ تا ۱۵ روز
  - 🔴 قرمز: کمتر از ۷ روز (یا گذشته)
- **انتقال دیتا به گوشی بعدی:** چون دیتا روی Firebase (ابر) ذخیره میشه نه روی گوشی، لازم نیست چیزی منتقل کنید — کافیه روی گوشی جدید هم همین APK رو نصب کنید و با ایمیل/رمز وارد بشید؛ همون دیتای مشترک رو می‌بینه.
- اگه بعداً خواستید نوتیفیکیشن پوش واقعی (حتی وقتی اپ بسته‌ست) اضافه کنید، به Firebase Cloud Messaging + یه Cloud Function روزانه نیاز دارید که یه قدم فنی جداست؛ بگید تا اون بخش رو هم اضافه کنم.
